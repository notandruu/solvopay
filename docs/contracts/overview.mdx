SolvoPay's on-chain layer is three contracts: a library, a per-session escrow, and a factory that deploys escrows.

## Contracts

| Contract | Purpose |
|----------|---------|
| `VoucherLib` | EIP-712 typed data hashing for the `Voucher` struct |
| `SessionEscrow` | Holds deposits, verifies vouchers, settles payments |
| `SessionFactory` | Deploys escrows, supplies USDC to Aave on session open |

## Dependencies

- OpenZeppelin v5: `EIP712`, `ECDSA`, `ReentrancyGuard`, `SafeERC20`
- Aave V3: `IPool` (supply, withdraw)
- Solidity 0.8.24, EVM version cancun

## Security properties

- **No admin keys**: contracts are immutable after deployment. There is no owner, no pause, no upgrade.
- **No reentrancy**: `ReentrancyGuard` on `settle` and `refund`.
- **Signature binding**: each escrow is the EIP-712 `verifyingContract`. Vouchers from one session cannot be replayed on another.
- **Nonce enforcement**: `settle` only accepts nonces strictly greater than `highestNonce`. The first submitted voucher is final.
- **Deposit cap**: `totalAuthorized` must be `<= deposit`. The contract can never pay out more than was locked.
- **Agent recovery**: `refund()` is always callable after `deadlineAt + 7 days` regardless of backend availability.
